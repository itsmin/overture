# Session Start

Load project context and present current state.

## Instructions

### 1. Metrics Validation
Read the CLAUDE.md Current Metrics table. Quick-check values against their listed sources of truth. Flag any discrepancies — don't silently accept stale numbers.

### 2. Documentation Health
Check the Documentation Reference table for staleness. Files not updated in >7 days get a warning. This is a signal, not a blocker — note it and move on.

### 3. Deferred Work
Read the Deferred Work section. Present any bumped items. Items deferred for many sessions should be flagged for re-evaluation — park them explicitly or promote them.

### 4. Pending Verifications
Read Session Progress for pending verification items from previous sessions. Surface anything that can be checked now.

### 5. Cross-Project Coordination
<!-- Remove this section if the project doesn't participate in coordination. -->
Read the working contract at `[path/to/contract]`. Check APPROVED WORK and IMPLEMENTATION STATUS for items requiring this project's implementation.

### 6. Work Queue
Present current priorities from the Work Queue section. Include the NEXT SESSION pointer if one exists.

### 7. Alignment
Ask: "Before diving in — does today's work advance [project]'s current goals, or is there something more pressing?"

Then: **"Session context loaded. Ready to work."**
